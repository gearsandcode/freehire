# AGENTS.md — design-system

The design system package for freehire's web frontend. Sibling to `web/`, linked via `file:../design-system`. Contains tokens, primitives, Storybook, and docs.

## What this is

- **Tokens:** W3C DTCG `.tokens.json` files in `tokens/`, built to CSS custom properties via Style Dictionary. Source of truth for all colors, spacing, typography, radius, shadow, motion, and z-index values. Consumed by `web/src/app.css` via `@import`.
- **Primitives:** Svelte 5 components in `src/` — the shared UI building blocks (Button, Badge, Input, Card, Dialog, Tabs, Tooltip, Alert, Chip, Avatar, EmptyState, Skeleton, Pagination, FormField, Table). `web/src/lib/ui/index.ts` is a thin re-export barrel.
- **Storybook:** One CSF story file per primitive in `src/`. `pnpm build-storybook` for static build.
- **Docs:** DSDS entity JSON in `docs/dsds/` (foundation/theme/component entities). Living inventories in `docs/`.

## Commands

```bash
pnpm install                          # install deps
pnpm build                            # build tokens (same as tokens:build)
pnpm tokens:build                     # Style Dictionary → dist/tokens-{light,dark}.css
pnpm storybook                        # dev server on :6006
pnpm build-storybook                  # static build → storybook-static/
pnpm validate:docs                    # (phase 06: validate DSDS JSON — placeholder)
pnpm test                             # (no tests yet)
```

## Conventions

### Tokens

- **DTCG format:** Every token uses `$type`, `$value`, `$description`. No SD v3 `value`/`type` without `$` prefix.
- **Source of truth:** `tokens/*.tokens.json`. Never edit `dist/*.css` by hand — it's generated output (committed because `file:` deps have no publish step).
- **Light + dark:** Every color token gets both light (in `color.tokens.json`) and dark (in `color-dark.tokens.json`) values. Non-color tokens (spacing, typography, etc.) are theme-agnostic — one value, included in both light and dark outputs.
- **Aliases:** Use DTCG references (`{brand-ring}`) for semantic aliases. These resolve per-platform in the build.
- **Naming:** Token names are flat (no group wrappers) so generated CSS var names match existing `app.css` conventions (`--background`, not `--color-background`). New token categories prefix with their group (`--spacing-1`, `--font-size-lg`).

### Primitives

- **Svelte 5 runes:** `$props()`, `$state()`, `$bindable()`, `$derived()`. No Svelte 4 `export let` or `on:` event syntax.
- **tailwind-variants:** Use `tv()` for multi-variant primitives (Button, Badge, Chip, Alert, Tabs). Export the variant function and types from the `module` script.
- **cn() utility:** Import from `./cn.js` (the package's own copy, not `$lib/utils`). All `class` props merge via `cn()`.
- **Shared prop vocab:** `size` (sm/md/lg), `variant` (primary/secondary/outline/ghost or default/destructive/brand), `class` (string, merged via cn), `children` (Snippet).
- **Accessibility:** Interactive primitives need keyboard support + ARIA. Dialog: Escape, backdrop click, `role=dialog`, `aria-modal`. Tabs: arrow-key nav, `role=tablist/tab/tabpanel`, `aria-selected`. Tooltip: `role=tooltip`, hover + focus trigger.
- **Tokens only:** No hardcoded color values (hex, rgb, oklch) in primitives. No Tailwind arbitrary values (`bg-[#...]`, `p-[7px]`). All values come from CSS custom properties generated by the token build.
- **No web/ coupling:** Primitives must not import from `$lib/` or depend on `web/`-specific stores/types. They're in a separate package.

### Stories

- **CSF format:** `.stories.ts` files (not `.stories.svelte` — the svelte-vite indexer doesn't handle those).
- **One file per primitive:** Cover all variants and states. Use `argTypes` with `control: 'select'` for variant props.
- **Autodocs:** Every story has `tags: ['autodocs']`.

### DSDS docs

- **Entity JSON:** `docs/dsds/foundation.json` (7 foundation entities), `docs/dsds/theme.json` (dark theme), `docs/dsds/components.json` (15 component entities).
- **Source linking:** Each entity links to its source file via `source.file` (relative to the JSON file).

## Architecture

```
design-system/
  package.json           name: freehire-design-system, exports: ./src/*, ./dist/*, ./tokens/*
  pnpm-workspace.yaml    pnpm 11 structural config (strictDepBuilds: false)
  sd.config.mjs           Style Dictionary config (unused — build-tokens.mjs replaces it)
  scripts/
    build-tokens.mjs     SD build script: two SD instances (light + dark), custom shadow/css transform
  tokens/
    color.tokens.json    23 semantic color tokens (light)
    color-dark.tokens.json  dark overrides for all 23 colors
    spacing.tokens.json  13-step spacing scale
    typography.tokens.json  fonts, sizes, weights, line heights
    radius.tokens.json   base + derivations
    shadow.tokens.json   4-step elevation
    motion.tokens.json   durations + easings
    z-index.tokens.json  6-step stacking scale
  src/
    cn.ts                cn() utility (clsx + tailwind-merge)
    index.ts             barrel export
    button.svelte        ...15 primitive .svelte files
    *.stories.ts         10 CSF story files
  dist/                  generated CSS (committed)
    tokens-light.css     :root { --background: ...; ... }
    tokens-dark.css      .dark { --background: ...; ... }
  docs/
    component-inventory.md   living doc: all web/ components classified
    icon-inventory.md         living doc: all icons in web/
    inventory-summary.md      phase 1 investigation summary
    dsds/
      foundation.json    7 foundation entities
      theme.json         dark theme entity
      components.json    15 component entities
  .storybook/
    main.ts              svelte-vite framework, viteFinal hook for Svelte plugin
    preview.ts           imports tokens-light.css, dark background toggle
```

## CI

The `design-system` CI job (`.github/workflows/ci.yml`) runs: `pnpm install` → `pnpm build` → `pnpm test` → `pnpm validate:docs` → `pnpm build-storybook`. The `web` job depends on `design-system` succeeding first.
